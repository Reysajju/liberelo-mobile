How to Fix It

Run this command in your PowerShell terminal:

PowerShell
ipconfig /flushdns
Once that completes, try running your build command again: npm run build:apk.

2. Switch to a Public DNS Server (Most Reliable Fix)
If flushing doesn't help, your local Internet Service Provider's (ISP) default DNS server is likely struggling to resolve Expo's domain. Switching to a stable public DNS like Google or Cloudflare typically solves this instantly.

Press Win + R, type ncpa.cpl, and hit Enter to open your Network Connections window.

Right-click your active internet connection (Wi-Fi or Ethernet) and select Properties.

Double-click Internet Protocol Version 4 (TCP/IPv4) from the list.

Preferred DNS server: 8.8.8.8 (Google) or 1.1.1.1 (Cloudflare)

Alternate DNS server: 8.8.4.4 or 1.0.0.1

Click OK to save, restart your PowerShell terminal to apply the changes, and re-run your build.

3. Check for VPNs, Proxies, or Firewalls
VPNs: If you have a VPN running, it might be misconfiguring your terminal's DNS routing. Try turning it off temporarily.

Firewall: Ensure that a local security suite or Windows Defender isn't explicitly blocking outbound Node.js network traffic.

Remove-Item -Recurse -Force node_modules, package-lock.json
(If you are using Git Bash or Mac/Linux instead of PowerShell, use: rm -rf node_modules package-lock.json)

2. Reinstall clean dependencies
PowerShell
npm install
3. Install the missing peer dependency & force the correct Babel version for SDK 54
PowerShell
npx expo install react-native-worklets babel-preset-expo
(Using npx expo install automatically forces both packages to match the exact versions required by Expo SDK 54).

4. Run the EAS build with a cleared cloud cache
PowerShell
npx eas-cli@latest build -p android --profile preview --clear-cache
